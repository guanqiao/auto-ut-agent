## 问题描述

LLM 配置没有被正确持久化和加载。每次应用启动时都创建空的配置集合，用户配置的 LLM 设置在重启后会丢失。

## 根本原因

1. `main_window.py` 第 300 行：每次启动时创建空的 `LLMConfigCollection()`
2. 没有保存 `LLMConfigCollection` 到文件的功能
3. 没有从文件加载 `LLMConfigCollection` 的功能
4. 对比 Aider 配置，`config.py` 中有 `save_aider_config()` 和 `load_aider_config()`，但缺少 LLM 配置的对应函数

## 修复步骤

### 1. 修改 `pyutagent/config.py`

* 添加 `get_llm_config_path()` 函数 - 获取 LLM 配置文件路径

* 添加 `save_llm_config(config_collection)` 函数 - 保存配置到 JSON 文件

* 添加 `load_llm_config()` 函数 - 从 JSON 文件加载配置

### 2. 修改 `pyutagent/ui/main_window.py`

* 导入 `load_llm_config` 和 `save_llm_config`

* 修改 `__init__`：启动时调用 `load_llm_config()` 加载已有配置

* 修改 `on_llm_config()`：配置更新后调用 `save_llm_config()` 保存到文件

### 3. 修改 `pyutagent/llm/config.py`（如需要）

* 确保 `LLMConfigCollection` 可以正确序列化和反序列化

* API Key 使用 `SecretStr`，已有 `@field_serializer` 处理序列化

## 涉及文件

1. `pyutagent/config.py` - 添加保存/加载函数
2. `pyutagent/ui/main_window.py` - 集成保存/加载逻辑

## 预期结果

* 用户配置的 LLM 配置会被保存到 `~/.pyutagent/llm_config.json`

* 应用重启后能自动加载之前的配置

* 配置包括多个 LLM 配置项和默认配置 ID
