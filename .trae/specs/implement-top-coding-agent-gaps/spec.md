# 顶级 Coding Agent 核心能力 Gap 补齐 - 需求规格说明书

## 1. 概述

### 1.1 项目背景
PyUT Agent 在任务规划、自主性、架构设计方面已接近顶级水平，但在用户体验和开发工具链集成方面存在关键差距。本项目旨在对标 Cursor、Trae、Qoder、Claude Code、OpenCode 等顶级 Coding Agent，补齐核心能力短板。

### 1.2 目标
- 3 个月内补齐 P0 级核心差距
- 6 个月内完善 P1 级重要能力
- 12 个月内优化 P2 级体验功能

### 1.3 范围
- P0：VS Code 插件深度集成、Diff 预览 + 审批流程、通用终端集成
- P1：Tab 代码补全、全代码库语义索引、多 Agent 深度协作、Skills 生态完善
- P2：代码风格学习、长期规划优化、云端执行/沙箱、PR 审查集成

---

## 2. 功能需求

### 2.1 P0 - 核心功能（必须立即补齐）

#### FR-P0-1: VS Code 插件深度集成

**描述**：完善现有 VS Code 插件，达到 Cursor 级别的用户体验

**用户故事**：
- 作为 Java 开发者，我希望在 VS Code 中直接使用 PyUT Agent，无需切换工具
- 作为开发者，我希望通过 Chat 面板自然语言对话生成测试
- 作为开发者，我希望在文件树右键菜单中快速生成测试

**功能需求**：
| ID | 功能描述 | 优先级 |
|----|---------|--------|
| FR-P0-1.1 | Chat 面板（Webview + React），支持自然语言对话 | 高 |
| FR-P0-1.2 | 文件树集成，右键菜单生成测试 | 高 |
| FR-P0-1.3 | Diff 预览组件，可视化代码变更 | 高 |
| FR-P0-1.4 | 终端面板集成，命令执行 | 高 |
| FR-P0-1.5 | 状态指示器，显示任务进度 | 高 |
| FR-P0-1.6 | 快捷键支持（Ctrl+G 生成测试） | 中 |
| FR-P0-1.7 | 主题定制，支持深色/浅色主题 | 中 |
| FR-P0-1.8 | 设置面板，配置 LLM、Maven 等 | 中 |

**验收标准**：
- [ ] Chat 面板可以正常发送和接收消息
- [ ] 右键菜单可以触发测试生成
- [ ] Diff 预览可以正确显示代码变更
- [ ] 终端可以执行任意命令
- [ ] 状态指示器实时更新任务进度
