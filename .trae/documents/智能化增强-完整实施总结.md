## 🎯 核心功能模块详解

### 1. **SemanticAnalyzer (代码语义分析器)**

**文件**: [`pyutagent/core/semantic_analyzer.py`](d:\opensource\github\auto-ut-agent\pyutagent\core\semantic_analyzer.py)

#### 核心能力
- ✅ **方法调用图构建**: 分析代码依赖关系
- ✅ **业务逻辑识别**: 7 种类型 (VALIDATION, TRANSFORMATION, PERSISTENCE, RETRIEVAL, CALCULATION, STATE_CHANGE, EXTERNAL_CALL)
- ✅ **数据流分析**: 追踪数据在方法间的流动
- ✅ **边界条件识别**: 6 种类型 (NULL_CHECK, EMPTY_CHECK, RANGE_CHECK, TYPE_CHECK, STATE_CHECK, PERMISSION_CHECK)
- ✅ **测试场景推导**: 自动生成正常、边界、异常场景
- ✅ **复杂度计算**: 基于参数和结构的圈复杂度

#### 数据结构
```python
CallGraph          # 方法调用关系图
BusinessLogic      # 业务逻辑片段
DataFlow           # 数据流信息
BoundaryCondition  # 边界条件
TestScenario       # 测试场景
CallGraphNode      # 调用图节点
```

#### 使用示例
```python
from pyutagent.core.semantic_analyzer import SemanticAnalyzer

analyzer = SemanticAnalyzer()
result = analyzer.analyze_file("UserService.java", java_class)

# 获取测试场景
scenarios = analyzer.get_test_scenarios()

# 获取调用图
call_graph = analyzer.get_call_graph()
```

#### 预期收益
- 📈 测试场景覆盖率提升 **30%**
- 📈 边界条件识别准确率提升 **50%**
- 📈 测试优先级自动排序

### 2. **RootCauseAnalyzer (错误根因分析器)**

**文件**: [`pyutagent/core/root_cause_analyzer.py`](d:\opensource\github\auto-ut-agent\pyutagent\core\root_cause_analyzer.py)

#### 核心能力
- ✅ **编译错误分析**: 12 种错误类别自动识别
- ✅ **测试失败分析**: 断言失败、异常捕获
- ✅ **根本原因定位**: 自动识别 contributing factors
- ✅ **修复策略推荐**: 14 种策略类型
- ✅ **置信度评估**: 0.0-1.0 评分
- ✅ **决策支持**: 是否重试、是否需要人工审查

#### 错误类别 (12 种)
1. **SYNTAX_ERROR** - 语法错误
2. **TYPE_ERROR** - 类型错误
3. **IMPORT_ERROR** - 导入错误
4. **REFERENCE_ERROR** - 引用错误
5. **ACCESS_ERROR** - 访问权限错误
6. **OVERRIDE_ERROR** - 重写错误
7. **INITIALIZATION_ERROR** - 初始化错误
8. **RESOURCE_ERROR** - 资源错误
9. **ASSERTION_ERROR** - 断言错误
10. **MOCK_ERROR** - Mock 错误
11. **NULL_POINTER_ERROR** - 空指针错误
12. **LOGIC_ERROR** - 逻辑错误

#### 修复策略 (14 种)
1. **SYNTAX_FIX** - 语法修复
2. **TYPE_CORRECTION** - 类型修正
3. **IMPORT_ADDITION** - 添加导入
4. **VISIBILITY_CHANGE** - 可见性修改
5. **SIGNATURE_UPDATE** - 签名更新
6. **INITIALIZATION_FIX** - 初始化修复
7. **RESOURCE_MANAGEMENT** - 资源管理修复
8. **ASSERTION_FIX** - 断言修复
9. **MOCK_SETUP** - Mock 设置
10. **NULL_CHECK** - 空值检查
11. **LOGIC_CORRECTION** - 逻辑修正
12. **CODE_REMOVAL** - 代码删除
13. **CODE_ADDITION** - 代码添加
14. **CODE_MODIFICATION** - 代码修改

#### 数据结构
```python
CompilationError    # 编译错误
TestFailure         # 测试失败
RootCause           # 根本原因
FixStrategy         # 修复策略
RootCauseAnalysis   # 根因分析结果
```

#### 使用示例
```python
from pyutagent.core.root_cause_analyzer import RootCauseAnalyzer

analyzer = RootCauseAnalyzer()

# 分析编译错误
analysis = analyzer.analyze_compilation_errors(compiler_output)

# 获取根本原因
root_causes = analysis.root_causes

# 获取推荐修复
suggested_fixes = analysis.suggested_fixes

# 置信度评估
confidence = analysis.confidence_score
```

#### 预期收益
- 📈 错误修复成功率提升 **40%**
- 📈 平均修复迭代次数减少 **50%**
- 📈 智能推荐最佳修复策略

### 3. **IntelligenceEnhancer (智能化增强器)**

**文件**: [`pyutagent/agent/intelligence_enhancer.py`](d:\opensource\github\auto-ut-agent\pyutagent\agent\intelligence_enhancer.py)

#### 核心能力
- ✅ **集成分析组件**: 统一接口调用 SemanticAnalyzer 和 RootCauseAnalyzer
- ✅ **增强提示词生成**: 将智能分析结果注入 LLM 提示词
- ✅ **智能决策支持**: 重试决策、优先级排序
- ✅ **结果缓存**: 提升性能，避免重复分析
- ✅ **场景过滤**: 按方法名、优先级过滤测试场景

#### 关键方法
```python
analyze_target_code()           # 分析目标代码
analyze_compilation_errors()    # 分析编译错误
analyze_test_failures()         # 分析测试失败
generate_enhanced_prompt()      # 生成增强提示词
get_priority_scenarios()        # 获取高优先级场景
should_retry_generation()       # 判断是否重试
```

#### 增强流水线
```
EnhancedGenerationPipeline:
  1. 语义分析 
  → 2. 测试场景识别 
  → 3. 智能生成 
  → 4. 错误分析 
  → 5. 智能修复
```

#### 使用示例
```python
from pyutagent.agent.intelligence_enhancer import IntelligenceEnhancer

enhancer = IntelligenceEnhancer()

# 分析目标代码
intelligence_result = enhancer.analyze_target_code(
    "UserService.java",
    java_class
)

# 生成增强提示词
enhanced_prompt = enhancer.generate_enhanced_prompt(
    base_prompt,
    intelligence_result
)

# 使用增强提示词调用 LLM
test_code = llm_client.generate(enhanced_prompt)
```

#### 预期收益
- 📈 测试生成质量提升 **35%**
- 📈 LLM 调用效率提升 (更精准的提示词)
- 📈 减少无效重试次数

### 4. **IntelligenceEnhancedCoT (智能化 CoT 引擎)**

**文件**: [`pyutagent/llm/intelligence_cot.py`](d:\opensource\github\auto-ut-agent\pyutagent\llm\intelligence_cot.py)

#### 核心能力
- ✅ **集成语义分析到 CoT**: 将业务逻辑、边界条件注入推理步骤
- ✅ **集成根因分析到修复**: 将错误分类、修复策略注入修复提示
- ✅ **动态推理步骤**: 根据分析结果生成针对性推理链
- ✅ **4 种增强提示模板**: 测试生成、错误修复、边界分析、业务逻辑

#### 增强提示模板

##### 1. semantic_enhanced_test_generation
```
特性:
- 业务逻辑类型识别

- 测试场景推荐

- 调用图信息

推理步骤:
1. ANALYZE - 审查语义分析结果
2. IDENTIFY - 识别测试场景
3. PLAN - 规划测试结构
4. GENERATE - 生成测试代码
5. VERIFY - 验证语义覆盖
```

##### 2. rca_enhanced_error_fix
```
特性:
- 错误类别和置信度
- 根本原因和证据
- 推荐修复策略
- 成功概率评估

推理步骤:
1. ANALYZE - 分析根因
2. UNDERSTAND - 理解上下文
3. IDENTIFY - 选择最佳策略
4. REFINE - 应用并验证
```

##### 3. boundary_enhanced_generation
```
特性:
- 参数边界条件矩阵
- 边界优先级排序
- 边界测试策略

推理步骤:
1. ANALYZE - 分析参数边界
2. IDENTIFY - 识别关键边界
3. PLAN - 规划边界测试
4. GENERATE - 生成边界测试
```

##### 4. business_logic_test_design
```
特性:
- 业务逻辑类型分布
- 前置/后置条件
- 副作用识别

推理步骤:
1. ANALYZE - 分析业务逻辑

3. PLAN - 规划业务测试
4. GENERATE - 生成业务测试
```

    # 请求人工审查
    request_human_review(analysis)
