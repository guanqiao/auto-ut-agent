# 持续分析和优化集成计划

## 背景

基于对代码库的分析，发现以下需要改进的地方：

### 当前状态

| 类别 | 状态 | 说明 |
|------|------|------|
| 集成测试 | ✅ 完善 | 6 个集成测试文件 |
| CI/CD | ❌ 缺失 | 无 GitHub Actions 配置 |
| 代码质量工具 | ❌ 缺失 | 无 ruff/mypy/black 配置 |
| 测试覆盖率 | ⚠️ 部分 | 有 pytest-cov，缺配置 |
| 性能优化 | ✅ 有计划 | ARCHITECTURE_OPTIMIZATION_PLAN.md |

## 目标

1. 添加 GitHub Actions CI/CD 配置
2. 配置代码质量工具（ruff, mypy）
3. 配置测试覆盖率报告和阈值
4. 集成性能基准测试到 CI

## 实现方案

### 任务 1：创建 GitHub Actions CI 配置

**新文件**: `.github/workflows/ci.yml`

**功能**:
- 触发条件：push 到 master、pull_request
- 测试矩阵：Python 3.9, 3.10, 3.11, 3.12
- 步骤：
  1. 检出代码
  2. 安装依赖
  3. 运行代码检查（ruff, mypy）
  4. 运行测试
  5. 生成覆盖率报告
  6. 上传覆盖率到 Codecov

### 任务 2：配置代码质量工具

**修改文件**: `pyproject.toml`

**添加配置**:

```toml
[tool.ruff]
line-length = 100
target-version = "py39"
select = ["E", "F", "W", "I", "N", "UP", "B", "C4"]
ignore = ["E501"]

[tool.ruff.per-file-ignores]
"tests/*" = ["S101"]

[tool.mypy]
python_version = "3.9"
warn_return_any = true
warn_unused_configs = true
ignore_missing_imports = true
```

### 任务 3：配置测试覆盖率

**修改文件**: `pyproject.toml`

**添加配置**:

```toml
[tool.coverage.run]
source = ["pyutagent"]
branch = true
omit = ["tests/*", "*/__pycache__/*", "*/ui/*"]

[tool.coverage.report]
exclude_lines = [
    "pragma: no cover",
    "def __repr__",
    "raise NotImplementedError",
    "if TYPE_CHECKING:",
    "if __name__ == .__main__.:",
]
fail_under = 60
```

### 任务 4：创建 PR 检查工作流

**新文件**: `.github/workflows/pr-check.yml`

**功能**:
- 代码格式检查
- 类型检查
- 测试运行
- 覆盖率检查

### 任务 5：创建发布工作流

**新文件**: `.github/workflows/release.yml`

**功能**:
- 自动发布到 PyPI
- 生成 GitHub Release
- 更新 CHANGELOG

| `.github/workflows/ci.yml` | 新增 | CI 工作流 |
| `.github/workflows/pr-check.yml` | 新增 | PR 检查工作流 |
| `.github/workflows/release.yml` | 新增 | 发布工作流 |

| `.gitignore` | 修改 | 添加覆盖率报告忽略 |

## 测试计划

1. 本地运行 ruff 检查
2. 本地运行 mypy 类型检查
3. 本地运行测试并生成覆盖率报告
4. 验证 CI 工作流语法

## 风险评估

- **兼容性风险**: ruff/mypy 可能报告现有代码问题
- **CI 时间**: 新增检查可能增加 CI 运行时间
- **覆盖率阈值**: 初始阈值设置过低可能失去意义

## 预期结果

1. 每次提交自动运行测试
2. 代码质量自动检查
3. 覆盖率报告自动生成
4. PR 合并前自动检查
