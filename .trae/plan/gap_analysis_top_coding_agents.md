## 三、核心能力差距分析

### 3.1 差距评估矩阵

| 能力维度 | 竞品水平 | 现状 | 差距等级 | 影响 |
|---------|---------|------|---------|------|
| **IDE 原生集成** | Cursor/Trae 重建编辑器内核 | PyQt6 独立应用 | ⭐⭐⭐⭐⭐ | P0 |
| **Diff 预览审批** | Cursor 逐行审批 | 无可视化 | ⭐⭐⭐⭐⭐ | P0 |
| **终端集成** | 通用终端+任意命令 | 预定义 Maven/Git | ⭐⭐⭐⭐ | P0 |
| **语义索引** | Cursor 全库语义 | 向量存储+KG | ⭐⭐⭐⭐ | P1 |
| **代码补全** | Cursor Tab 级别 | 无 | ⭐⭐⭐⭐⭐ | P1 |
| **Agent Skills** | OpenCode/Claude 标准 | 无 | ⭐⭐⭐ | P1 |
| **计划模式** | Cursor 澄清性问题 | 基本规划 | ⭐⭐⭐ | P1 |
| **多形态产品** | Trae IDE+插件双形态 | 分散 | ⭐⭐⭐ | P1 |
| **模型路由** | Qoder 自动选择 | 手动配置 | ⭐⭐⭐ | P2 |
| **代码审查** | Cursor 内嵌审查 | 无 | ⭐⭐⭐ | P2 |

### 3.2 详细差距分析

#### Gap 1: IDE 原生集成 ⭐⭐⭐⭐⭐ (P0)

**对标**: Cursor（重建编辑器）、Trae（双形态）

**现状差距**:
1. ❌ 无原生 IDE（仅 PyQt6 独立应用）
2. ❌ VS Code 插件功能基础
3. ❌ 无法在代码编辑器中直接使用
4. ❌ 用户需离开主开发环境

**影响**:
- 用户体验差
- 采用率低
- 难以与 Cursor/Trae 竞争

---

#### Gap 2: Diff 预览和审批 ⭐⭐⭐⭐⭐ (P0)

**对标**: Cursor（逐行审批）、Cline（自主/手动模式）

**现状差距**:
1. ❌ 无可视化 Diff UI
2. ❌ 无审批模式切换
3. ❌ 无一键回滚机制

**影响**:
- 信任度低
- 控制力弱
- 风险高

---

#### Gap 3: 终端集成 ⭐⭐⭐⭐ (P0)

**对标**: Cursor（内置终端）、Claude Code（CLI）

**现状差距**:
1. ❌ 仅预定义 Maven/Git 命令
2. ❌ 无通用终端组件
3. ❌ 无法执行任意命令

**影响**:
- 灵活性差
- 调试困难
- 能力受限

---

#### Gap 4: 代码补全 ⭐⭐⭐⭐⭐ (P1)

**对标**: Cursor Tab、Copilot

**现状差距**:
1. ❌ 无实时代码补全
2. ❌ 无 Tab 建议功能
3. ❌ 无法在编辑时提供建议

**影响**:
- 无法与 Copilot 竞争
- 用户需额外工具

---

#### Gap 5: Agent Skills 标准 ⭐⭐⭐ (P1)

**对标**: Anthropic Agent Skills、OpenCode

**现状差距**:
1. ❌ 无 Agent Skills 支持
2. ❌ 无标准化工具定义
3. ❌ 无法与其他工具生态互操作

**影响**:
- 生态兼容差
- 扩展性受限

---

#### Gap 6: 语义代码理解 ⭐⭐⭐⭐ (P1)

**对标**: Cursor 全代码库索引

**现状差距**:
1. ⚠️ 向量存储和知识图谱存在
2. ⚠️ 但未深度集成到编辑体验
3. ⚠️ 语义检索能力有限

**影响**:
- 代码理解深度不足
- 跨文件修改质量受限

---

#### Gap 7: 计划模式 ⭐⭐⭐ (P1)

**对标**: Cursor（澄清性问题优化计划）

**现状差距**:
1. ⚠️ 基本任务规划存在
2. ❌ 无澄清性提问
3. ❌ 无迭代优化

**影响**:
- 复杂任务规划质量低
